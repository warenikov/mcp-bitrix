# mcp-bitrix

MCP сервер для управления Битриксом через Claude. Позволяет создавать инфоблоки, управлять элементами, настраивать highload-блоки, работать с пользователями и многое другое — прямо из чата с Claude.

## Требования

- PHP 8.1+
- Composer
- Битрикс CMS (любая версия с поддержкой D7)
- Docker (для текущей версии)

## Установка

### 1. Установите пакет в корне вашего Битрикс-проекта

```bash
composer require warenikov/mcp-bitrix
```

### 2. Убедитесь что в PHP-контейнере есть Composer

Если используете Docker, добавьте в `Dockerfile`:

```dockerfile
COPY --from=composer:latest /usr/bin/composer /usr/bin/composer
```

Пересоберите образ:

```bash
docker compose build php && docker compose up -d php
```

### 3. Добавьте сервер в `.mcp.json` вашего проекта

Создайте (или дополните) файл `.mcp.json` в корне проекта:

```json
{
  "mcpServers": {
    "bitrix": {
      "command": "docker",
      "args": ["exec", "-i", "ИМЯ_КОНТЕЙНЕРА", "php", "/var/www/html/vendor/bin/server"]
    }
  }
}
```

Замените `ИМЯ_КОНТЕЙНЕРА` на имя вашего PHP-контейнера. Узнать его можно командой:

```bash
docker ps --format "{{.Names}}"
```

### 4. Перезапустите Claude Code

После добавления `.mcp.json` перезапустите Claude Code или выполните `/mcp` чтобы подхватить новый сервер.

## Использование

После подключения Claude видит все инструменты и может управлять Битриксом в диалоге:

> «Создай инфоблок "Каталог товаров" с кодом `catalog` типа `content`»

> «Добавь свойство "Цена" типа число к инфоблоку с ID 3»

> «Покажи все элементы инфоблока "Новости" за последний месяц»

## Реализованные инструменты

### Типы инфоблоков
| Инструмент | Описание |
|---|---|
| `list_iblock_types` | Список всех типов |
| `create_iblock_type` | Создать тип |
| `delete_iblock_type` | Удалить тип |

### Инфоблоки
| Инструмент | Описание |
|---|---|
| `list_iblocks` | Список инфоблоков (фильтр по типу) |
| `get_iblock` | Получить по ID |
| `create_iblock` | Создать |
| `update_iblock` | Обновить |
| `delete_iblock` | Удалить |

### Свойства инфоблоков
| Инструмент | Описание |
|---|---|
| `list_iblock_properties` | Список свойств |
| `add_iblock_property` | Добавить свойство (S/N/L/F/G/E) |
| `update_iblock_property` | Обновить |
| `delete_iblock_property` | Удалить |

### Секции
| Инструмент | Описание |
|---|---|
| `list_iblock_sections` | Список секций |
| `add_iblock_section` | Добавить |
| `update_iblock_section` | Обновить |
| `delete_iblock_section` | Удалить |

### Элементы
| Инструмент | Описание |
|---|---|
| `get_iblock_elements` | Выборка с фильтром и постраничкой |
| `get_iblock_element` | Получить по ID (со свойствами) |
| `add_iblock_element` | Добавить |
| `update_iblock_element` | Обновить (поля + свойства) |
| `delete_iblock_element` | Удалить |

## Как это работает

```
Claude → MCP protocol (stdio) → docker exec -i {container} php vendor/bin/server → Bitrix PHP API → JSON
```

Сервер запускается внутри Docker-контейнера через `docker exec`. PHP-процесс инициализирует ядро Битрикса и напрямую вызывает API (`CIBlock`, `CIBlockElement` и т.д.). Никаких внешних зависимостей — MCP-протокол реализован напрямую.
